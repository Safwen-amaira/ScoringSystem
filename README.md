# Threat Recommendation Engine

Web-based AI-assisted recommendation and scoring system for SOC workflows. Paste raw Wazuh alerts, MISP events, and Cortex analysis JSON directly into the API or GUI, and the service will normalize the data, extract IOCs, calculate PKIs, score the threat, and generate recommendations plus HTML email content.

## What this MVP does

- Accepts evidence from Wazuh, MISP, and Cortex in a single API request
- Returns a score from `0` to `100`
- Produces a workflow decision:
  - `continue`
  - `review`
  - `stop`
- Accepts raw source JSON directly
- Extracts IOCs automatically
- Calculates PKIs automatically
- Returns `allow_workflow_to_continue` for fast orchestration gating
- Returns score JSON for workflow orchestration
- Returns email content JSON with HTML and text bodies
- Uses local Ollama or file-based agents instead of API-key providers
- Offers a web GUI for analysts
- Runs as a Docker container on Ubuntu

## Architecture

- `FastAPI` backend exposes:
  - `POST /api/score`
  - `POST /api/score/raw`
  - `POST /api/email-content`
  - `POST /api/analyze`
  - `POST /api/recommendation`
  - `POST /api/recommendation/email`
  - `POST /api/score-and-email`
  - `GET /health`
- Static frontend in `web/` is served by the same container
- SMTP configuration is optional and controlled by environment variables
- AI generation is local-first and controlled by environment variables

## Endpoint contracts

### `POST /api/score`

Returns workflow gating JSON:

```json
{
  "score": 87,
  "decision": "stop",
  "allow_workflow_to_continue": false,
  "summary": "High-confidence malicious activity was observed on a sensitive asset.",
  "ai_generated": true,
  "ai_provider": "ollama",
  "breakdown": [],
  "evidence": []
}
```

### `POST /api/email-content`

Returns email JSON with HTML:

```json
{
  "subject": "[STOP] Security recommendation for srv-finance-01 (87/100)",
  "html": "<html><body>...</body></html>",
  "text": "Security recommendation for workflow: ...",
  "ai_generated": true,
  "ai_provider": "ollama"
}
```

### `POST /api/score-and-email`

Returns both payloads together and optionally sends the mail if `analyst_email` and SMTP settings are present.

### `POST /api/score/raw`

Accepts raw Wazuh/MISP/Cortex JSON and returns the scoring-only response.

### `POST /api/recommendation`

Accepts raw Wazuh/MISP/Cortex JSON and returns the full recommendation response, including:

- normalized score
- workflow decision
- recommendation text
- extracted IOCs
- calculated PKIs

## AI behavior

- The score and workflow gate remain deterministic so the orchestration decision is immediate.
- Recommendation text and email HTML can be generated by local Ollama.
- If Ollama is unavailable, file-based prompt agents are used as a local fallback.
- Agent prompt files live in `agents/security_summary.md` and `agents/email_html.md`.
- The stack auto-starts Ollama and auto-pulls the configured model during `docker compose up`.

Note:
- This repository uses prompt-specialized local agents, not true model fine-tuning. It is deployment-ready without extra setup, but it does not perform heavyweight training during startup.

AI environment variables:

- `AI_PROVIDER` values: `ollama` or `file-agent`
- `OLLAMA_BASE_URL` default: `http://ollama:11434`
- `OLLAMA_MODEL` default: `llama3.1:8b`
- `AI_TIMEOUT_SECONDS` default: `30`

## Run locally

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Open [http://localhost:8000](http://localhost:8000).

## Run with Docker

```bash
docker compose up --build -d
```

Open [http://YOUR_SERVER_IP:8000](http://YOUR_SERVER_IP:8000).

## Ubuntu deployment

1. Install Docker and Docker Compose plugin on the Ubuntu server.
2. Copy this project to the server.
3. Clone the repository.
4. Enter the project directory.
5. Start the stack:

```bash
docker compose up
```

6. Wait for Ollama model download on first startup.
7. Call the raw JSON endpoints.

That is the intended deployment flow.
